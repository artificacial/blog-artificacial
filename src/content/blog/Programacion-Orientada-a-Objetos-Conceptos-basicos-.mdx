# Programación Orientada a Objetos: Fundamentos y Principios Básicos

La Programación Orientada a Objetos (POO) es un paradigma de programación que se basa en el concepto de "objetos", los cuales representan entidades del mundo real. A través de la POO, los programas se organizan de manera que las operaciones y los datos que operan en ellos están agrupados en unidades llamadas objetos. Este enfoque ofrece una forma más intuitiva y estructurada de modelar y resolver problemas complejos.

## 1. Conceptos Fundamentales de la Programación Orientada a Objetos

### 1.1. Objetos

Un objeto es una instancia particular de una clase, que representa una entidad concreta dentro de un programa. Por ejemplo, si estamos construyendo un programa para una biblioteca, podríamos tener una clase llamada `Libro`, y cada libro específico sería un objeto de esa clase.

```javascript
class Libro {
	constructor(titulo, autor) {
		this.titulo = titulo
		this.autor = autor
	}
}

// Creación de objetos
let libro1 = new Libro('El Quijote', 'Miguel de Cervantes')
let libro2 = new Libro('Harry Potter', 'J.K. Rowling')

console.log(libro1.titulo) // Output: El Quijote
console.log(libro2.autor) // Output: J.K. Rowling
```

Este código en JavaScript utiliza la sintaxis de clases introducida en ECMAScript 6 para definir la clase Libro. Luego, los objetos libro1 y libro2 se crean utilizando el operador new, de manera similar a la creación de instancias en Python. Las propiedades del objeto se acceden utilizando la notación de punto (libro1.titulo, libro2.autor).

### 1.2. Clases

Una clase es un plano, un modelo o una plantilla para crear objetos. Define los atributos y métodos que todos los objetos de esa clase compartirán. En nuestro ejemplo anterior, `Libro` es una clase que define la estructura de los libros.

### 1.3. Atributos

Los atributos son variables que están asociadas a un objeto particular. Representan las características o propiedades de ese objeto. En el ejemplo anterior, `titulo` y `autor` son atributos de la clase `Libro`.

### 1.4. Métodos

Los métodos son funciones definidas dentro de una clase que pueden ser llamadas para realizar operaciones en los objetos de esa clase. Estos métodos pueden utilizarse para modificar los atributos de un objeto o para realizar acciones relacionadas con él.

```javascript
class Libro {
	constructor(titulo, autor) {
		this.titulo = titulo
		this.autor = autor
	}

	mostrarInformacion() {
		console.log('Título:', this.titulo)
		console.log('Autor:', this.autor)
	}
}

let libro1 = new Libro('El Quijote', 'Miguel de Cervantes')
libro1.mostrarInformacion()
// Output:
// Título: El Quijote
// Autor: Miguel de Cervantes
```

En JavaScript, la función **init** se reemplaza con el método constructor. Además, el método mostrar_informacion se convierte en mostrarInformacion, utilizando la notación de camelCase, común en JavaScript. La impresión de información se realiza con console.log() en lugar de print(). Por último, para llamar al método, se utiliza la notación de punto (libro1.mostrarInformacion()).

### 1.5. Encapsulación

La encapsulación es un concepto que permite ocultar la implementación interna de un objeto y solo exponer las partes necesarias para interactuar con él. Esto se logra mediante el uso de métodos de acceso (getters) y métodos de modificación (setters).

```javascript
class Libro {
	constructor(titulo, autor) {
		this.__titulo = titulo
		this.__autor = autor
	}

	get titulo() {
		return this.__titulo
	}

	set titulo(titulo) {
		this.__titulo = titulo
	}
}

let libro1 = new Libro('El Quijote', 'Miguel de Cervantes')
console.log(libro1.titulo) // Output: El Quijote
libro1.titulo = 'Don Quijote de la Mancha'
console.log(libro1.titulo) // Output: Don Quijote de la Mancha
```

En JavaScript, utilizamos los getters y setters para obtener y establecer los valores de las propiedades de manera segura. Los getters y setters se definen utilizando la sintaxis de métodos especiales get y set, y pueden ser llamados directamente como si fueran propiedades.

### 1.6. Herencia

La herencia es un mecanismo que permite que una clase (llamada clase derivada o subclase) herede atributos y métodos de otra clase (llamada clase base o superclase). Esto fomenta la reutilización de código y permite la creación de jerarquías de clases.

```javascript
class Animal {
	constructor(nombre) {
		this.nombre = nombre
	}

	hacerSonido() {
		// Método abstracto, debe ser implementado por las subclases
	}
}

class Perro extends Animal {
	hacerSonido() {
		return '¡Guau!'
	}
}

class Gato extends Animal {
	hacerSonido() {
		return '¡Miau!'
	}
}

let perro = new Perro('Bobby')
let gato = new Gato('Mittens')

console.log(perro.hacerSonido()) // Output: ¡Guau!
console.log(gato.hacerSonido()) // Output: ¡Miau!
```

En este ejemplo, `Animal` es la clase base y `Perro` y `Gato` son clases derivadas que heredan de `Animal`. Cada clase derivada redefine el método `hacer_sonido()` para que sea específico de esa clase.En JavaScript, las clases y métodos se definen de manera similar a Python, pero con algunas diferencias de sintaxis. Aquí, se define una clase base Animal con un método hacerSonido() que actúa como un método abstracto. Luego, se crean las subclases Perro y Gato que heredan de Animal y sobrescriben el método hacerSonido() con su propio comportamiento. Finalmente, se crean instancias de Perro y Gato y se llama al método hacerSonido() para imprimir los sonidos respectivos.

### 1.7. Polimorfismo

El polimorfismo es la capacidad que tienen los objetos de diferentes clases de responder al mismo mensaje de manera distinta. Esto permite tratar objetos de diferentes clases de manera uniforme.

```javascript
class Figura {
	area() {
		// Método abstracto, debe ser implementado por las subclases
	}
}

class Cuadrado extends Figura {
	constructor(lado) {
		super()
		this.lado = lado
	}

	area() {
		return this.lado ** 2
	}
}

class Circulo extends Figura {
	constructor(radio) {
		super()
		this.radio = radio
	}

	area() {
		return 3.14159 * this.radio ** 2
	}
}

let figuras = [new Cuadrado(4), new Circulo(3)]

for (let figura of figuras) {
	console.log('Área:', figura.area())
}
```

En JavaScript, las clases y métodos se definen de manera similar a Python, pero con algunas diferencias de sintaxis. En este código, Figura es una clase base con un método area() que actúa como un método abstracto. Las subclases Cuadrado y Circulo heredan de Figura y sobrescriben el método area() con su propia implementación. Luego, se crean instancias de Cuadrado y Circulo y se las itera para imprimir el área de cada figura.

## 2. Principios Básicos de la Programación Orientada a Objetos

### 2.1. Abstracción

La abstracción es el proceso de identificar las características esenciales de un objeto y eliminar los detalles irrelevantes. En POO, las clases y los objetos son formas de abstracción, ya que representan entidades del mundo real de manera simplificada.

### 2.2. Encapsulamiento

El encapsulamiento implica ocultar los detalles internos de un objeto y exponer solo las partes necesarias para interactuar con él. Esto se logra definiendo atributos como privados y proporcionando métodos de acceso y modificación para trabajar con esos atributos.

### 2.3. Herencia

La herencia permite que una clase herede atributos y métodos de otra clase. Esto fomenta la reutilización de código y la organización jerárquica de clases.

### 2.4. Polimorfismo

El polimorfismo permite que los objetos de diferentes clases respondan al mismo mensaje de manera distinta. Esto facilita el diseño de programas flexibles y extensibles.

## 3. Conclusiones

La Programación Orientada a Objetos es un paradigma poderoso que permite modelar problemas de manera más intuitiva y estructurada. Al entender los conceptos fundamentales como objetos, clases, atributos, métodos, encapsulamiento, herencia y polimorfismo, los programadores pueden construir sistemas más eficientes, mantenibles y escalables.
